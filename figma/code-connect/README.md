# Figma Code Connect — Dashbook canonical 10

Code Connect maps the Dashbook Figma component sets to their `@dashfi/svelte`
code, so Figma **Dev Mode** shows the real import + props instead of
autogenerated CSS. This folder holds the authored templates for the canonical
10 components.

- **Figma file:** `91csAF1OGUmCZROdZlCRSv` (Dashbook library)
- **Template:** [`dashbook.figma.ts`](./dashbook.figma.ts) — one `figma.connect()`
  block per canonical component.

## Status — authored, not yet published

Three things must be true before these activate in Figma. Today, none are:

| Requirement | State | Notes |
| ----------- | ----- | ----- |
| Figma plan = Organization or Enterprise | ❌ Professional | Code Connect is not available on Free/Professional. `figma connect publish` will reject. |
| Library published to a team library | ❌ not yet | Publishing is a Figma-UI action (Plugin API cannot publish). |
| `@figma/code-connect` installed + configured | ❌ not yet | Add the dep + a `figma.config.json` where the lib lives. |

So these files are **ready-to-publish templates**, not live mappings. They are
correct against the current node IDs and the `@dashfi/svelte` prop APIs.

## How to publish (once the gate clears)

1. Upgrade the Figma team to Organization/Enterprise.
2. Publish the Dashbook library from the Figma UI (Assets → Publish).
3. Move `dashbook.figma.ts` next to the component source in
   `core/libs/svelte-components/lib/` (Code Connect files live with the code).
4. `pnpm add -D @figma/code-connect` and add `figma.config.json` (parser:
   `html` or the Svelte parser, per the installed version).
5. Verify the import specifier + `example` syntax against the installed CLI —
   Svelte support has shifted across releases.
6. `npx figma connect publish --token <FIGMA_TOKEN>`.
