- PR #16 says "New Sidebar component (**60 → 61**)", "**61** live previews",
  "**61**-component stability matrix", and the registry header

  **pre-#17** main. Main already registers `PaginationWrapper`, so the true base

- **Rule:** derive every count from `pnpm spec-audit` + the MCP list tool at PR

### Marketing provenance gate (§7.3)

`MarketingPatternSpec` (`src/lib/specs/marketing/patterns/types.ts`) has `source:
string` + optional `sourceNote` only. It lacks §7.3 provenance: source
commit/revision, `lastVerifiedAt`, and `verificationStatus`
(`verified|partially-verified|unverified|stale`). The five new marketing patterns
use freeform `source`/`sourceNote` strings. Landing new marketing patterns
without structured provenance is a §7.3 violation — split PR (4) is gated on the
type extension.

### Browser testing gate (primary surface)

PR #16's own verification says live browser checks were **blocked by a port
conflict**; only svelte-check + build (SSR) ran. Stateful product patterns and
the marketing pattern routes must be browser-tested before merge, and their
MCP/JSON/skill output validated (§10.2 — a change that renders in the browser but
breaks MCP/JSON/skill output is not done).
